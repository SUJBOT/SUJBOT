## SEVERITY GUIDELINES

- **critical**: Service completely down, security breach suspected, disk full (>90%), SSL expired or <7 days, database connection lost
- **warning**: High resource usage (>80%), elevated error count (>10/period), SSL <30 days, container restarts, failed login spike
- **info**: Normal operational notes, minor anomalies, informational items

## OVERALL STATUS RULES

- `critical` if ANY critical finding exists
- `warning` if ANY warning finding exists (and no critical)
- `healthy` if only info findings or no findings
